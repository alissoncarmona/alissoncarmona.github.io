Introduction to Cybersecurity

Q&A to discover cybersecurity’s world

.1 What does the “CIA” triade stands for ?

This acronym stands for :

  • Confidentiality (making sure your data is secured and accessible by only people who are meant to access it)
  • Integrity (making sure the data can be trusted)
  • Availability (data and ressources must be accessible to individuals)

.2 What is the difference between exploit and vulnerability ?

A vulnerability is a weakness, a flaw in a security system. (To picture this idea, imagine that you leave your door unlocked)
“Exploit”, here, means that someone will use this vulnerability against you. They will do an action, an attack, by exploiting the vulnerability they found (For example, intrude your house because you didn’t lock your door)

.3 How do you tell if a file has a virus ?

You can :

  1. Draw your file on a specific website to identify known malwares.
  2. Compare the hash of the file with known good hashs.
  3. Use a reverse engineering tool.
    Bonus: Open the file in an isolated VM and check if it’s corrupted or not.

.4 What is a brute force attack and how do we prevent it ?

A brute force attack consists of trying to find your password with an automated script which contains a list of commonly used passwords.
What can you do ?

  • Create a long and complex password, with special characters.
  • Set account lock-out (after too many attempts, their account will be lock out for a specific duration)
  • Take the IP address of the attacker and ban it. It would also be safer to change your password just in case.

.5 What is SQL injection attack and how do we prevent it ?

Attackers exploit input fields on websites (which lack meant sanitization) and inject malicious SQL code to execute. They use it to read, edit or event delete informations of the website’s database.
What can be done ?

  • Input validation : making sure that what users type in the input field matches what’s expected.
  • Parameterized queries : making sure the input is text and not code.
